# DevSpark V2 产品构想（2026-02 更新）

## 1. 市场调研与竞品验证（更新版）

### 竞品分层（验证结论）

1. **超级阅读/情报平台**：Feedly、Inoreader 已将“AI 摘要 + 去重/聚类 + 批量报告”作为核心能力，解决阅读效率与信息噪音问题。
2. **消费型 AI 新闻 App**：Particle 强调“Story 聚类 + 可定制摘要风格”，主打阅读体验与视角平衡。
3. **搜索型 AI**：Perplexity 以主动搜索与长文输出为主，但长文能力处在迭代调整阶段。

**结论**：市场对“读”与“搜”的 AI 化已经充分，但**从阅读到创作发布的闭环工作流仍缺口明显**。

### 竞品对比矩阵（V2 版）

| 维度     | DevSpark（目标）                     | Feedly / Inoreader        | Perplexity      | Particle     |
| -------- | ------------------------------------ | ------------------------- | --------------- | ------------ |
| 核心场景 | **阅读 → 思考 → 写作闭环**           | 监控/阅读/摘要/报告       | 主动搜索 + 问答 | 纯阅读消费   |
| 信息源   | **自定义 RSS + 社区 API + 主题订阅** | RSS/关键词为主            | 全网搜索        | 算法推荐     |
| 输出能力 | **意图模板 + 证据包 + 可发布草稿**   | 摘要/报告（非写作工作流） | 长文输出        | 摘要/梗概    |
| 中文社区 | **强支持（V2EX/知乎/公众号/掘金）**  | 弱                        | 一般            | 弱           |
| 目标用户 | **技术创作者/DevRel/研究员**         | 企业情报/高频阅读         | 通用大众        | 碎片阅读人群 |

## 2. 产品定位（V2 版）

**定位一句话**：面向中文开发者与科技内容创作者的“读写一体化工作台”，帮助用户将信息流直接转化为**可发布成果**。

**核心价值主张**：

- 降噪与聚合：把分散信息整理为高信噪比“主题包”。
- 即时创作：从“看到新闻”到“生成可发布草稿”不超过 3 步。
- 可信与可追溯：每条结论都有来源证据，可追溯、可引用。

**Slogan**：Stop Scrolling, Start Shipping

---

## 3. 目标用户画像（细化）

1. **技术内容创作者**：每天需要选题、产出、复盘。
2. **独立开发者 / 个人创业者**：需要跟进竞品和趋势，快速输出观点。
3. **技术管理者 / DevRel**：需要生成周报、分享行业动态。
4. **持续学习者**：时间有限，但希望跟上技术变化。

---

## 4. 关键机会点（V2 强化）

1. **“摘要”升级为“证据包”**：不仅告诉你发生了什么，还提供关键证据、来源与争议点。
2. **“灵感库”升级为“主题空间”**：把碎片化阅读沉淀成可持续迭代的主题资产。
3. **“一键写作”升级为“意图驱动写作流”**：从风格、长度、受众和渠道角度定义输出。
4. **“阅读工具”升级为“发布工具”**：内置发布格式与导出通道，减少上下文切换。

## 5. V2 功能方案（结构化模块）

### 5.1 智能聚合引擎（Ingestion & Governance）

- **多源接入**：RSS / HN / V2EX / GitHub Trending / 公众号 RSS 等。
- **去重 + 聚类**：事件级合并，保留不同视角，避免信息重复。
- **信噪评分**：从“技术深度、讨论热度、来源可信度”综合评分。
- **合规策略**：仅抓取用户自有订阅内容，保留原文链接。
- 可支持opml导入

### 5.2 认知处理中心（Cognitive Core）

- **结构化摘要卡片**：
- **主题空间（Topic Spaces）**：每个主题自动累积时间线、关键节点与高质量引用。
- **观点融合**：将用户的零碎想法与系统摘要融合成草稿框架。

### 5.3 创作与互动工作台（Creative Workspace）

- **双栏读写模式**：左读右写，点击“引用”即可插入证据与观点。
- **意图驱动写作模板**：
  - 快速推文/微博
  - 技术博客大纲
  - 周报/内参
  - 竞品分析
- **灵感卡片 2.0**：支持“系列化写作建议”，自动提示潜在选题。

### 5.4 分发与增长（Publish & Growth）

- **一键导出**：Markdown / Notion / Obsidian。
- **渠道预设**：适配公众号、X、即刻、小红书的格式化输出。
- **发布后回收**：把发布后的反馈与数据回流到主题空间。

## 6. MVP 与阶段路线图

**MVP（8-12 周）**

- RSS + 社区源接入
- 去重/聚类 + 结构化摘要
- 读写双栏 + 2 个模板（推文/博客大纲）
- 导出 Markdown

**V1.5（3-6 个月）**

- 主题空间 + 灵感库
- 周报自动生成
- 多渠道格式化输出

**V2（6-12 个月）**

- 团队协作与共享主题空间
- 证据包/观点融合强化
- API / Webhook 生态

---

## 7. 关键指标（OKR 方向）

- 阅读 → 首稿时间（TTFD: Time to First Draft）
- 周产出数（Drafts/Week）
- 主题空间留存率（Topic Retention）
- 去重率与信噪比提升（Noise Reduction %）

---

## 8. 技术架构建议（沿用并优化）

- **后端**：Java 21 + Spring Boot 3.4
- **AI 编排**：Spring AI（可切换 OpenAI / Claude / 本地模型）
- **数据层**：PostgreSQL + pgvector
- **任务调度**：Virtual Threads + 定时抓取
- **前端**：Next.js / React（偏写作工作台）
- **部署**：Docker + 可插拔向量库

## 9. 商业化（V2 方案）

- **Free**：10 个订阅源，日限量摘要
- **Pro**：无限订阅 + 读写模板 + 导出
- **Team**：共享主题空间 + 协作报告
